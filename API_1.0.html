1 通用定义

  1. 时间定义
    除特殊说明之外，所有时间通信使用北京时间通信。格式为14位yyyyMMddhhmmss ,例如20141225140000，如果业务无需精确到秒，根据业务需要后面填充最小/最大时间。
    注：客户端需要自行处理时区问题，与服务器的通信均认为是北京时间。
  2. session管理
    使用无状态会话，客房端访问通过 Headers参数:
    Authorization: "eyJhbGciOiJIUzUxMiJ9.eyJzdWIiOiIxODUwMDAzMDkyOCIsImp0aSI6IjcxIiwidHlwZSI6MTB9.ThINc4S1f6EiRsDDqgHn67c3WM-Es9UtczK69bRaPoD2aJyljj3jmh6TAMoov1vqUv9gtopG7qpyzV6nJvRA"。

  3. 错误信息
    除下载文件外，所有返回信息均为json，必定包含success属性，有错误时必定包含messageCode属性

    {
    "success": false,
    "data": { },
    "messageCode": null,
    "message: "失败!"
}

  4. 提交方式
    全用非标准REST架构: 数据修改(POST), 数据获取(GET)

  5. 请求报文url及参数格式
    http://${ip}:${port}/api/业务系统/业务场景
    所有url全部使用小写英文字母
    示例:
    http://${ip}:${port}/api/login/
